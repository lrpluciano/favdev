Java EE (Jakarta EE)

Plataforma para desenvolvimento de aplicações empresariais em Java

Arquitetura Java EE

Java EE (agora Jakarta EE) é uma plataforma robusta para desenvolvimento de aplicações empresariais distribuídas. Ela oferece uma arquitetura completa com diversos containers e componentes que facilitam o desenvolvimento de sistemas complexos.

Nota do Professor

Alunos, pensem na plataforma Jakarta EE como um ecossistema completo que fornece tudo o que vocês precisam para construir aplicações corporativas escaláveis. Ela abstrai muitas complexidades, permitindo que vocês se concentrem na lógica de negócio em vez de infraestrutura.

A plataforma Jakarta EE é composta por quatro containers principais:

  • Applet Container: Executa applets Java no navegador. (Menos utilizado atualmente)
  • Application Client Container: Executa aplicações cliente no computador do usuário.
  • Web Container: Executa componentes web como Servlets, JSP e Server Faces.
  • Enterprise Beans Container: Executa componentes de negócio (Enterprise Beans).

Ponto Chave

Todos esses containers permitem o desenvolvimento de aplicações escaláveis, com otimização de recursos e suporte a transações distribuídas. A plataforma segue o padrão MVC (Model-View-Controller), promovendo a separação de responsabilidades.

Como funciona?

Quando um usuário faz uma requisição, o servidor web direciona para o servlet correspondente. O servlet processa a requisição, gera o conteúdo HTML e envia como resposta. Tudo isso acontece no servidor, antes de enviar a página para o navegador do cliente.

Os servlets são mapeados para URLs através de anotações ou arquivos de configuração (Deployment Descriptors). Eles são o fundamento de todas as outras tecnologias web em Java EE.

Java Server Pages (JSP): Permite inserir código Java em HTML, facilitando a separação entre lógica de negócio e apresentação.

<html> <body> <h1>Lista de Filmes</h1> <ul> <c:forEach var="m" items="${movies}"> <li>${m.title}</li> </c:forEach> </ul> </body> </html>

Nota do Professor

JSP é uma evolução natural dos servlets. Em vez de escrever código HTML dentro de Java (como nos servlets), nós escrevemos código Java dentro de HTML. Isso torna o código mais limpo e mais fácil para designers web trabalharem. Mas lembrem-se: o JSP é compilado para um servlet antes de ser executado!

JSP utiliza tag libraries (JSTL) e expressões EL (Expression Language) para simplificar o acesso a dados e objetos Java. Isso reduz significativamente a quantidade de código Java necessário nas páginas.

Server Faces: Framework MVC baseado em componentes para construção de interfaces de usuário. Oferece gerenciamento de estado, validação e navegação.

<h:form> <h:inputText value="#{hello.name}" /> <h:commandButton value="Submit" action="response" /> </h:form>

Server Faces simplifica o desenvolvimento de interfaces complexas, fornecendo componentes reutilizáveis e um ciclo de vida bem definido. Ele gerencia automaticamente o estado da aplicação entre requisições, algo que seria complexo de implementar manualmente com servlets ou JSP.

Enterprise Beans e Persistência

Os Enterprise Beans são componentes que contêm lógica de negócio e são executados no Enterprise Beans Container. Eles são o coração da camada de negócio em aplicações Java EE.

Nota do Professor

Enterprise Beans são como especialistas em tarefas específicas da sua aplicação. Eles sabem como lidar com transações, segurança, concorrência e outros aspectos complexos do desenvolvimento empresarial. Vocês podem se concentrar na regra de negócio enquanto o container cuida do resto!

Session Beans: Implementam a lógica de negócio da aplicação. Podem ser stateless (sem estado) ou stateful (com estado).

Característica Stateless Session Bean Stateful Session Bean
Estado Não mantém estado entre chamadas Mantém estado específico para cada cliente
Escalabilidade Mais escalável, pode ser compartilhado Menos escalável, dedicado a um cliente
Uso típico Serviços genéricos, cálculos Carrinhos de compras, sessões de usuário
Performance Melhor performance, menor consumo de recursos Maior consumo de recursos
@Stateless public class CalculadoraService { public double somar(double a, double b) { return a + b; } public double subtrair(double a, double b) { return a - b; } } @Stateful public class CarrinhoCompras { private List itens = new ArrayList<>(); public void adicionarItem(Item item) { itens.add(item); } public List getItens() { return itens; } public double getTotal() { return itens.stream().mapToDouble(Item::getPreco).sum(); } }

Quando usar cada um?

Use Stateless Session Beans para operações que não dependem de estado anterior, como cálculos, consultas ou serviços genéricos. Use Stateful Session Beans quando precisar manter conversação com o cliente, como em um carrinho de compras ou um assistente de configuração.

Entity Beans: Representam dados persistentes, geralmente mapeados para tabelas de banco de dados.

@Entity public class Cliente implements Serializable { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @Column(nullable = false) private String nome; @Column(unique = true, nullable = false) private String email; @Column(nullable = false) private String senha; // Getters e setters }

Entity Beans são gerenciados pelo container e oferecem funcionalidades como cache, lazy loading e transações. As anotações JPA (@Entity, @Id, @Column, etc.) definem como a classe Java será mapeada para a tabela do banco de dados.

Jakarta Persistence API (JPA): API para mapeamento objeto-relacional, permitindo que objetos Java sejam persistidos em bancos de dados relacionais.

Nota do Professor

JPA é uma das tecnologias mais importantes do Java EE. Ela permite que vocês trabalhem com objetos Java em vez de SQL direto, o que torna o código mais limpo e orientado a objetos. A implementação mais popular do JPA é o Hibernate, mas existem outras como EclipseLink e OpenJPA.

@Entity @Table(name = "produtos") public class Produto implements Serializable { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @Column(nullable = false, length = 100) private String nome; @Column(nullable = false, precision = 10, scale = 2) private BigDecimal preco; @ManyToOne @JoinColumn(name = "categoria_id") private Categoria categoria; // Getters e setters } // Exemplo de uso do EntityManager @Stateless public class ProdutoService { @PersistenceContext private EntityManager em; public void salvar(Produto produto) { em.persist(produto); } public Produto buscarPorId(Long id) { return em.find(Produto.class, id); } public List listarTodos() { return em.createQuery("SELECT p FROM Produto p", Produto.class) .getResultList(); } public void atualizar(Produto produto) { em.merge(produto); } public void excluir(Long id) { Produto produto = em.find(Produto.class, id); if (produto != null) { em.remove(produto); } } }

Vantagens do JPA

  • Produtividade: Reduz significativamente a quantidade de código boilerplate para acesso a dados.
  • Manutenção: Código mais limpo e orientado a objetos.
  • Portabilidade: Mesmo código funciona com diferentes bancos de dados.
  • Cache: Melhora de performance através de mecanismos de cache.
  • Transações: Gerenciamento automático de transações.

Frameworks JavaScript

Principais frameworks e bibliotecas para desenvolvimento web com JavaScript

Nota do Professor

JavaScript evoluiu muito nos últimos anos! Hoje, não é mais apenas uma linguagem para adicionar interatividade simples às páginas. Com os frameworks modernos, podemos construir aplicações complexas e robustas tanto no front-end quanto no back-end. Vamos explorar as tecnologias mais importantes.

Ajax

Ajax (Asynchronous JavaScript and XML) é uma técnica para criar aplicações web interativas, permitindo comunicação assíncrona com o servidor sem recarregar a página.

function loadDoc() { var xhttp = new XMLHttpRequest(); xhttp.onreadystatechange = function() { if (this.readyState == 4 && this.status == 200) { document.getElementById("demo").innerHTML = this.responseText; } }; xhttp.open("GET", "conteudo_remoto.txt", true); xhttp.send(); }

Principais características:

  • Atualização parcial de páginas
  • Comunicação assíncrona com o servidor
  • Melhora na experiência do usuário

Node.js

Node.js é um ambiente de execução JavaScript server-side, baseado na engine V8 do Google Chrome. Permite criar servidores web e aplicações de rede.

var http = require('http'); const hostname = '127.0.0.1'; const port = 3000; const server = http.createServer((req, res) => { res.statusCode = 200; res.setHeader('Content-Type', 'text/plain'); res.end('Hello World\n'); }); server.listen(port, hostname, () => { console.log(`Server running at http://${hostname}:${port}/`); });

Principais características:

  • Modelo assíncrono e orientado a eventos
  • Ecossistema rico com npm (Node Package Manager)
  • Execução JavaScript no servidor

Angular

Angular é um framework para desenvolvimento de aplicações web front-end, mantido pelo Google. É baseado em TypeScript e utiliza o padrão MVC.

import { Component } from '@angular/core'; @Component({ selector: 'app-hello', template: `<h1>{{ message }}</h1>` }) export class HelloComponent { message = 'Hello, Angular!'; }

Principais características:

  • Baseado em componentes
  • Injeção de dependência
  • Binding de dados
  • Diretivas para manipulação do DOM

Angular CLI

A Angular CLI é uma ferramenta de linha de comando para criar, construir e manter aplicações Angular. Ela automatiza muitas tarefas repetitivas e segue as melhores práticas do framework.

Nota do Professor

A CLI é sua melhor amiga no desenvolvimento Angular! Ela não apenas cria a estrutura de arquivos correta, mas também configura o ambiente de desenvolvimento, otimiza a construção para produção e ajuda a manter o código organizado. Usem sempre a CLI para criar componentes, serviços, módulos e outros elementos da aplicação.

Para instalar globalmente:

npm install -g @angular/cli

Para criar um novo projeto:

ng new meu-projeto

Comandos essenciais da CLI

  • ng generate component nome - Cria um novo componente

  • ng test - Executa testes unitários

Componentes

Componentes são os blocos de construção fundamentais das aplicações Angular. Um componente é composto por:

  • Uma classe TypeScript com propriedades e métodos
  • Um template HTML
  • Estilos CSS

Nota do Professor

Pensem nos componentes como "blocos LEGO" da sua aplicação. Cada componente tem uma responsabilidade específica e pode ser reutilizado em diferentes partes da aplicação. Essa abordagem baseada em componentes torna o código mais modular, mais fácil de testar e mais fácil de manter.

Para criar um componente com a CLI:

ng generate component MeuComponente

Exemplo de componente:

import { Component } from '@angular/core'; @Component({ selector: 'app-meu-componente', templateUrl: './meu-componente.component.html', styleUrls: ['./meu-componente.component.css'] }) export class MeuComponenteComponent { titulo = 'Meu Componente'; descricao = 'Este é um exemplo de componente Angular'; exibirMensagem() { alert('Olá do meu componente!'); } }

O decorador @Component define os metadados do componente, incluindo o seletor CSS, o template e os estilos. O seletor é usado para incluir o componente em outros templates HTML.

Boas práticas com componentes

  • Single Responsibility: Cada componente deve ter uma única responsabilidade bem definida.
  • Reutilização: Projete componentes para serem reutilizáveis em diferentes contextos.
  • Imutabilidade: Evite modificar diretamente as propriedades de entrada (@Input).
  • Desacoplamento: Use serviços e injeção de dependência para desacoplar lógica de negócio.

Binding

Angular oferece várias formas de binding para conectar o template com a classe do componente. O binding é o mecanismo que permite a comunicação entre o TypeScript e o HTML.

Nota do Professor

O binding é um dos conceitos mais poderosos do Angular! Ele permite que vocês criem interfaces dinâmicas e reativas sem precisar manipular diretamente o DOM. Pensem no binding como uma "ponte" entre a lógica da aplicação (TypeScript) e a apresentação (HTML).

Interpolação
Property Binding
Event Binding
Two-way Binding

Interpolação: Exibe valores de propriedades do componente no template usando a sintaxe {{ }}. É a forma mais simples de binding.

<!-- Exemplo de interpolação --> <h1>{{ titulo }}</h1> <p>Total: {{ 1 + 1 }}</p> <p>Data: {{ dataAtual | date:'short' }}</p> <p>{{ usuario.nome }} tem {{ usuario.idade }} anos</p>

Quando usar interpolação?

Use interpolação para exibir dados dinâmicos no template, como textos, números, datas e resultados de expressões simples. É ideal para conteúdo que será apenas exibido, não modificado.

Nota do Professor

A interpolação é ótima para exibir dados, mas lembrem-se: ela não funciona para todos os atributos HTML. Por exemplo, para definir o valor de um atributo como src ou href, vocês precisam usar property binding, que veremos a seguir.

Property Binding: Define o valor de propriedades de elementos do DOM ou diretivas usando a sintaxe [ ]. É usado para passar valores do componente para propriedades de elementos HTML.

<!-- Exemplos de property binding --> <img [src]="imageUrl"> <div [hidden]="isHidden">Conteúdo</div> <button [disabled]="isSubmitting">Enviar</button> <div [class.active]="isActive">Item ativo</div> <div [style.color]="textColor">Texto colorido</div>

Diferença entre property binding e interpolação

Enquanto a interpolação é usada para exibir conteúdo dentro de elementos, o property binding é usado para definir valores de atributos de elementos. Por exemplo, você usaria interpolação para o texto de um parágrafo (<p>{{ texto }}</p>), mas property binding para a fonte de uma imagem (<img [src]="urlImagem">).

Event Binding: Responde a eventos do DOM (como cliques, entradas de teclado, etc.) usando a sintaxe ( ). É usado para passar eventos do template para o componente.

<!-- Exemplos de event binding --> <button (click)="salvar()">Salvar</button> <input (keyup)="onKeyUp($event)"> <input (keyup.enter)="onEnter()"> <div (mouseover)="showTooltip()">Passe o mouse</div> <form (submit)="onSubmit($event)">

Objeto $event

Quando você passa $event como parâmetro para o método do componente, Angular fornece o objeto de evento DOM. Isso permite acessar informações como o valor de um input, a tecla pressionada, as coordenadas do mouse, etc.

Nota do Professor

O event binding é essencial para criar interfaces interativas. Ele permite que vocês respondam às ações do usuário e atualizem o estado da aplicação. Lembrem-se de manter os métodos do componente simples e delegar lógica complexa para serviços quando necessário.

Two-way Binding: Combina property binding e event binding usando a sintaxe [( )], também conhecida como "banana in a box". Permite a sincronização bidirecional de dados entre o template e o componente.

<!-- Exemplo de two-way binding --> <input [(ngModel)]="nome"> <p>Olá, {{ nome }}!</p> <!-- Equivalente a --> <input [value]="nome" (input)="nome = $event.target.value">

Configuração necessária

Para usar two-way binding com ngModel, você precisa importar o FormsModule no seu módulo:

import { NgModule } from '@angular/core'; import { BrowserModule } from '@angular/platform-browser'; import { FormsModule } from '@angular/forms'; @NgModule({ imports: [ BrowserModule, FormsModule // Importe o FormsModule aqui ], // ... }) export class AppModule { }

Nota do Professor

O two-way binding é muito útil para formulários, onde você precisa que os dados sejam atualizados em ambas as direções. No entanto, usem com moderação! Em aplicações maiores, é preferível usar property binding e event binding separadamente para ter mais controle sobre o fluxo de dados.

Diretivas

Diretivas são instruções no DOM. Existem três tipos principais de diretivas em Angular:

  • Componentes: Diretivas com template.
  • Diretivas Estruturais: Alteram o layout do DOM. Exemplos: *ngIf, *ngFor.
  • Diretivas de Atributo: Alteram a aparência ou comportamento de um elemento. Exemplo: ngClass, ngStyle.

Nota do Professor

Diretivas são como "extensões" do HTML! Elas permitem que vocês criem comportamentos personalizados e reutilizem lógica em diferentes partes da aplicação. Pensem nelas como uma forma de ensinar novos "truques" ao HTML.

*ngIf
*ngFor
ngClass
Diretivas Personalizadas

*ngIf: Adiciona ou remove elementos do DOM com base em uma condição booleana. É uma diretiva estrutural, por isso o asterisco (*) no início.

<!-- Exemplos de *ngIf --> <div *ngIf="mostrar">Este elemento será exibido apenas se 'mostrar' for true</div> <!-- Com else --> <div *ngIf="usuarioLogado; else loginTemplate"> Bem-vindo, {{ usuario.nome }}! </div> <ng-template #loginTemplate> <button>Fazer Login</button> </ng-template> <!-- Com then e else --> <div *ngIf="idade >= 18; then adulto; else menor"></div> <ng-template #adulto>Você é maior de idade</ng-template> <ng-template #menor>Você é menor de idade</ng-template>

Diferença entre *ngIf e [hidden]

Enquanto *ngIf adiciona ou remove elementos do DOM, [hidden] apenas os oculta visualmente (com display: none). Use *ngIf quando o elemento não deve existir na página (para melhorar performance) e [hidden] quando você precisa apenas ocultá-lo temporariamente.

*ngFor: Itera sobre uma lista e renderiza elementos para cada item. Também é uma diretiva estrutural.

<!-- Exemplos de *ngFor --> <ul> <li *ngFor="let item of itens">{{ item.nome }}</li> </ul> <!-- Com index --> <ul> <li *ngFor="let item of itens; let i = index"> {{ i + 1 }}. {{ item.nome }} </li> </ul> <!-- Com first, last, even, odd --> <div *ngFor="let item of itens; let first = first; let last = last; let even = even; let odd = odd" [class.first]="first" [class.last]="last" [class.even]="even" [class.odd]="odd"> {{ item.nome }} </div> <!-- Com trackBy para melhorar performance --> <ul> <li *ngFor="let item of itens; trackBy: trackById">{{ item.nome }}</li> </ul> // No componente trackById(index: number, item: any): number { return item.id; // ou qualquer identificador único }

Nota do Professor

O trackBy é muito importante para performance em listas grandes! Sem ele, Angular recria todos os elementos da lista sempre que houver uma mudança, mesmo que apenas um item tenha sido alterado. Com trackBy, Angular pode identificar quais itens realmente mudaram e atualizar apenas eles.

ngClass: Adiciona ou remove classes CSS dinamicamente com base em expressões. É uma diretiva de atributo.

<!-- Exemplos de ngClass --> <!-- String --> <div [ngClass]="'classe1 classe2'">Conteúdo</div> <!-- Array --> <div [ngClass]="['classe1', 'classe2']">Conteúdo</div> <!-- Object --> <div [ngClass]="{ 'active': isActive, 'error': hasError }">Conteúdo</div> <!-- Exemplo prático --> <div [ngClass]="{ 'card': true, 'card-primary': tipo === 'primary', 'card-success': tipo === 'success', 'card-danger': tipo === 'danger' }"> Conteúdo do card </div>

ngClass vs ngStyle

Enquanto ngClass manipula classes CSS, ngStyle manipula estilos inline. Use ngClass para estilos mais complexos e reutilizáveis (definidos em arquivos CSS) e ngStyle para estilos dinâmicos simples que não valem a pena definir em classes separadas.

Diretivas Personalizadas: Além das diretivas embutidas, você pode criar suas próprias diretivas para encapsular comportamentos reutilizáveis.

// Exemplo de diretiva de atributo personalizada import { Directive, ElementRef, HostListener, Input } from '@angular/core'; @Directive({ selector: '[appHighlight]' }) export class HighlightDirective { @Input() appHighlight = ''; constructor(private el: ElementRef) { } @HostListener('mouseenter') onMouseEnter() { this.highlight(this.appHighlight || 'yellow'); } @HostListener('mouseleave') onMouseLeave() { this.highlight(''); } private highlight(color: string) { this.el.nativeElement.style.backgroundColor = color; } } // Uso da diretiva <div appHighlight="lightblue">Passe o mouse sobre mim</div>

Nota do Professor

Diretivas personalizadas são uma forma poderosa de encapsular comportamentos que você precisa reutilizar em vários componentes. Em vez de repetir o mesmo código em múltiplos componentes, você cria uma diretiva e a aplica onde necessário. Isso segue o princípio DRY (Don't Repeat Yourself).

Serviços e Injeção de Dependência

Serviços são classes que encapsulam lógica de negócio que pode ser reutilizada em vários componentes. A injeção de dependência (DI) é um padrão de projeto usado para fornecer dependências a uma classe.

Nota do Professor

A injeção de dependência é um dos conceitos mais importantes do Angular! Ela permite que vocês criem código desacoplado, mais fácil de testar e mais fácil de manter. Pensem na DI como um "entregador" que leva as dependências (serviços, etc.) para as classes que precisam delas, em vez de as classes criarem suas próprias dependências.

Exemplo de serviço:

import { Injectable } from '@angular/core'; import { HttpClient } from '@angular/common/http'; import { Observable } from 'rxjs'; @Injectable({ providedIn: 'root' }) export class DataService { private apiUrl = 'https://api.example.com/data'; constructor(private http: HttpClient) { } obterDados(): Observable { return this.http.get(this.apiUrl); } obterPorId(id: number): Observable { return this.http.get(`${this.apiUrl}/${id}`); } criar(dado: any): Observable { return this.http.post(this.apiUrl, dado); } atualizar(id: number, dado: any): Observable { return this.http.put(`${this.apiUrl}/${id}`, dado); } excluir(id: number): Observable { return this.http.delete(`${this.apiUrl}/${id}`); } }

Para usar o serviço em um componente:

import { Component, OnInit } from '@angular/core'; import { DataService } from './data.service'; @Component({ selector: 'app-meu-componente', templateUrl: './meu-componente.component.html' }) export class MeuComponenteComponent implements OnInit { dados: any[]; constructor(private dataService: DataService) { } ngOnInit() { this.carregarDados(); } carregarDados() { this.dataService.obterDados().subscribe( (response) => { this.dados = response; }, (error) => { console.error('Erro ao carregar dados', error); } ); } }

Vantagens da Injeção de Dependência

  • Desacoplamento: Componentes não precisam saber como criar seus serviços.
  • Testabilidade: Fácil de substituir serviços por mocks em testes unitários.
  • Reutilização: Serviços podem ser compartilhados entre múltiplos componentes.
  • Manutenção: Centraliza a lógica de negócio em serviços, facilitando manutenção.

Nota do Professor

Uma boa prática é manter os componentes "leves", focados na apresentação, e delegar a lógica de negócio para serviços. Isso torna seus componentes mais fáceis de entender, testar e reutilizar. Lembrem-se: componentes são para apresentação, serviços são para lógica!

Módulos

Módulos (NgModule) organizam a aplicação em blocos coesos de funcionalidade. Todo aplicativo Angular tem pelo menos um módulo, o módulo raiz (AppModule).

Nota do Professor

Módulos são como "contêineres" que agrupam componentes, diretivas, pipes e serviços relacionados. Eles ajudam a organizar aplicações grandes, definem limites claros e permitem o lazy loading (carregamento sob demanda) para melhorar a performance inicial da aplicação.

Exemplo de módulo:

import { NgModule } from '@angular/core'; import { BrowserModule } from '@angular/platform-browser'; import { HttpClientModule } from '@angular/common/http'; import { FormsModule } from '@angular/forms'; import { RouterModule } from '@angular/router'; import { AppComponent } from './app.component'; import { MeuComponenteComponent } from './meu-componente/meu-componente.component'; import { OutroComponenteComponent } from './outro-componente/outro-componente.component'; import { DataService } from './data.service'; @NgModule({ declarations: [ AppComponent, MeuComponenteComponent, OutroComponenteComponent ], imports: [ BrowserModule, HttpClientModule, FormsModule, RouterModule.forRoot([ { path: '', component: MeuComponenteComponent }, { path: 'outro', component: OutroComponenteComponent } ]) ], providers: [ DataService ], bootstrap: [AppComponent] }) export class AppModule { }

O decorador @NgModule define o módulo com as seguintes propriedades:

  • declarations: Componentes, diretivas e pipes que pertencem ao módulo.
  • imports: Outros módulos cujas classes são necessárias para os componentes do módulo atual.
  • providers: Serviços que o módulo disponibiliza para a injeção de dependência.
  • bootstrap: Componente raiz que o Angular cria e insere no index.html.

Feature Modules e Lazy Loading

Para aplicações maiores, você pode criar feature modules para organizar funcionalidades específicas. Além disso, você pode configurar o lazy loading para que esses módulos sejam carregados apenas quando necessários, melhorando a performance inicial da aplicação.

// Exemplo de lazy loading no roteamento const routes: Routes = [ { path: 'admin', loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule) } ];

Ciclo de Vida dos Componentes

Os componentes têm um ciclo de vida gerenciado pelo Angular. Os hooks mais importantes são:

  • ngOnInit: Chamado após a inicialização das propriedades do componente. Ideal para inicializações.
  • ngOnDestroy: Chamado antes da destruição do componente. Ideal para limpeza de recursos.

Nota do Professor

Entender o ciclo de vida dos componentes é fundamental para controlar quando certas operações devem acontecer. Por exemplo, você não pode acessar elementos do DOM no construtor, mas pode no ngOnInit. E você deve cancelar inscrições em Observables no ngOnDestroy para evitar memory leaks!

Exemplo:

import { Component, OnInit, OnDestroy } from '@angular/core'; import { Subscription } from 'rxjs'; import { DataService } from './data.service'; @Component({ selector: 'app-meu-componente', templateUrl: './meu-componente.component.html' }) export class MeuComponenteComponent implements OnInit, OnDestroy { dados: any[]; private subscription: Subscription; constructor(private dataService: DataService) { console.log('Construtor chamado'); // Não é recomendado fazer chamadas HTTP aqui } ngOnInit() { console.log('ngOnInit chamado'); // Ideal para inicializações e chamadas a serviços this.subscription = this.dataService.obterDados().subscribe( (response) => { this.dados = response; } ); } ngOnDestroy() { console.log('ngOnDestroy chamado'); // Importante para limpar recursos e evitar memory leaks if (this.subscription) { this.subscription.unsubscribe(); } } }

Quando usar cada hook?

  • constructor(): Injeção de dependências e inicialização de propriedades simples.
  • ngOnChanges(): Reagir a mudanças em propriedades @Input.
  • ngOnInit(): Inicializações complexas, chamadas a serviços, acesso a elementos do DOM.
  • ngDoCheck(): Lógica de detecção de mudanças personalizada (use com cuidado).
  • ngOnDestroy(): Limpeza de recursos, cancelamento de inscrições, remoção de event listeners.

Roteamento

O roteamento permite navegar entre diferentes views da aplicação. O módulo RouterModule é responsável por essa funcionalidade.

Nota do Professor

O roteamento é essencial para criar Single Page Applications (SPAs). Ele permite que vocês tenham múltiplas "páginas" em uma aplicação de página única, sem recarregar o navegador inteiro. Isso proporciona uma experiência de usuário mais fluida e rápida.

Exemplo de configuração de rotas:

import { NgModule } from '@angular/core'; import { RouterModule, Routes } from '@angular/router'; import { HomeComponent } from './home/home.component'; import { SobreComponent } from './sobre/sobre.component'; import { ContatoComponent } from './contato/contato.component'; import { PageNotFoundComponent } from './page-not-found/page-not-found.component'; const routes: Routes = [ { path: '', component: HomeComponent }, { path: 'sobre', component: SobreComponent }, { path: 'contato', component: ContatoComponent }, { path: '**', component: PageNotFoundComponent } // Rota curinga ]; @NgModule({ imports: [RouterModule.forRoot(routes)], exports: [RouterModule] }) export class AppRoutingModule { }

Para navegar programaticamente:

import { Router } from '@angular/router'; export class MeuComponenteComponent { constructor(private router: Router) { } irParaSobre() { this.router.navigate(['/sobre']); } irParaContato() { this.router.navigate(['/contato']); } irParaComParametros() { this.router.navigate(['/produto', 123]); } }

Para exibir o componente correspondente à rota atual, use a diretiva <router-outlet></router-outlet> no template:

<!-- No app.component.html --> <header> <nav> <a routerLink="/" routerLinkActive="active">Home</a> <a routerLink="/sobre" routerLinkActive="active">Sobre</a> <a routerLink="/contato" routerLinkActive="active">Contato</a> </nav> </header> <main> <router-outlet></router-outlet> </main> <footer> <p>© 2025 Minha Aplicação</p> </footer>

Recursos avançados de roteamento

  • Rotas com parâmetros: { path: 'produto/:id', component: ProdutoComponent }
  • Rotas filhas: Permitem aninhar componentes dentro de outros.
  • Guardas de rota: Controlam o acesso às rotas (CanActivate, CanDeactivate, etc.).
  • Resolvers: Pré-carregam dados antes de ativar uma rota.

Pipes

Pipes são usados para transformar dados no template. Angular oferece vários pipes embutidos (Date, UpperCase, LowerCase, Currency, etc.) e permite criar pipes personalizados.

Nota do Professor

Pipes são como "funções de transformação" que você pode aplicar diretamente no template. Eles são muito úteis para formatar dados sem modificar a fonte original. Por exemplo, você pode exibir uma data em diferentes formatos, converter texto para maiúsculas, ou formatar números como moeda.

Exemplo de uso de pipe:

<!-- Pipes embutidos --> <p>{{ data | date:'short' }}</p> <p>{{ texto | uppercase }}</p> <p>{{ valor | currency:'BRL' }}</p> <p>{{ texto | slice:0:10 }}</p> <p>{{ array | json }}</p> <!-- Encadeando pipes --> <p>{{ data | date:'fullDate' | uppercase }}</p> <!-- Pipes com parâmetros --> <p>{{ numero | number:'1.2-2' }}</p> <p>{{ texto | truncate:20 }}</p>

Async Pipe

O async pipe é usado para lidar com Observables e Promises no template. Ele automaticamente se inscreve no Observable e atualiza o template quando novos valores são emitidos.

<!-- No componente --> export class MeuComponenteComponent { dados$ = this.meuService.obterDadosAsync(); constructor(private meuService: MeuService) { } } <!-- No template --> <!-- Com async pipe --> <div *ngIf="dados$ | async as dados"> <p>{{ dados.nome }}</p> <p>{{ dados.valor | currency }}</p> </div> <!-- Equivalente sem async pipe --> <div *ngIf="dados"> <p>{{ dados.nome }}</p> <p>{{ dados.valor | currency }}</p> </div> <!-- E no componente --> export class MeuComponenteComponent implements OnInit, OnDestroy { dados: any; private subscription: Subscription; constructor(private meuService: MeuService) { } ngOnInit() { this.subscription = this.meuService.obterDadosAsync().subscribe( response => this.dados = response ); } ngOnDestroy() { if (this.subscription) { this.subscription.unsubscribe(); } } }

Vantagens do Async Pipe

  • Simplicidade: Reduz significativamente a quantidade de código necessário.
  • Gerenciamento automático: Lida automaticamente com inscrição e cancelamento de inscrição.
  • Performance: Evita memory leaks e otimiza o uso de recursos.
  • Reatividade: Atualiza automaticamente o template quando novos valores são emitidos.

Criando Pipes Personalizados

Você pode criar seus próprios pipes para transformações específicas:

import { Pipe, PipeTransform } from '@angular/core'; @Pipe({ name: 'truncate' }) export class TruncatePipe implements PipeTransform { transform(value: string, limit = 25, completeWords = false, ellipsis = '...'): string { if (!value) return ''; if (value.length <= limit) { return value; } if (completeWords) { limit = value.substr(0, limit).lastIndexOf(' '); } return `${value.substr(0, limit)}${ellipsis}`; } } // Uso do pipe personalizado <p>{{ textoLongo | truncate:30 }}</p> <p>{{ textoLongo | truncate:30:true }}</p>

Nota do Professor

Pipes personalizados são uma ótima maneira de encapsular lógica de formatação que você precisa reutilizar em vários lugares da aplicação. Em vez de repetir a mesma lógica de transformação em múltiplos componentes, você cria um pipe e o aplica onde necessário.

Teste seus conhecimentos sobre Angular com base nos exercícios da Aula Atividade Tutor

Nota do Professor

Este quiz foi preparado para ajudar vocês a testarem seus conhecimentos sobre Angular e se prepararem para os exercícios da Aula Atividade Tutor. Tentem responder cada pergunta antes de verificar a resposta correta. Se errarem, leiam atentamente a explicação para entender o conceito.

1. Qual é o comando para criar um novo componente Angular usando a CLI?

  • ng new component MeuComponente
  • ng generate component MeuComponente
  • angular create component MeuComponente
  • npm create component MeuComponente
  • ng add component MeuComponente

Explicação: O Angular CLI usa o comando ng generate component (ou sua forma abreviada ng g c) para criar novos componentes. A alternativa correta é a segunda opção. O comando ng new é usado para criar um novo projeto Angular, não um componente.

2. Qual decorador é usado para definir um componente Angular?

  • @Component
  • @NgModule
  • @Directive
  • @Injectable
  • @Pipe

Explicação: O decorador @Component é usado para definir uma classe como um componente Angular, incluindo metadados como selector e templateUrl. @NgModule é usado para definir módulos, @Directive para diretivas, @Injectable para serviços e @Pipe para pipes.

3. Como você faz binding de uma propriedade em Angular?

  • {{propriedade}}
  • [propriedade]="valor"
  • (propriedade)="valor"
  • *ngIf="propriedade"
  • #propriedade

Explicação: O binding de propriedade usa colchetes [] para passar dados do componente para o template. Isso é conhecido como property binding. A sintaxe {{ }} é usada para interpolação, () para event binding, *ngIf é uma diretiva estrutural e # é usado para referências de template.

4. Qual diretiva é usada para renderizar elementos condicionalmente?

  • *ngFor
  • *ngIf
  • [ngSwitch]
  • [ngClass]
  • *ngWhile

Explicação: A diretiva *ngIf adiciona ou remove elementos do DOM com base em uma condição booleana, permitindo a renderização condicional de elementos. *ngFor é usada para iterar sobre listas, ngSwitch para múltiplas condições, ngClass para manipular classes CSS e não existe uma diretiva *ngWhile nativa no Angular.

5. Qual serviço é usado para navegação entre rotas?

  • Router
  • Route
  • Navigation
  • HttpClient
  • Location

Explicação: O serviço Router do pacote @angular/router é usado para gerenciar a navegação programática entre rotas na aplicação. O método navigate() é usado para navegar para uma rota específica. Route é usado para definir configurações de rota, HttpClient para requisições HTTP e Location para interagir com a URL do navegador.

6. Como você captura eventos no template?

  • (evento)="metodo()"
  • [evento]="metodo()"
  • {{evento}}
  • #evento
  • *ngEvent="metodo()"

Explicação: Parênteses () são usados para binding de eventos como (click) ou (keyup), permitindo capturar eventos do DOM e executar métodos do componente. A sintaxe [] é para property binding, {{ }} para interpolação, # para referências de template e não existe uma diretiva *ngEvent nativa.

7. Qual é a finalidade do ngOnInit()?

  • Inicializar propriedades do componente
  • Lidar com destruição do componente
  • Atualizar o DOM
  • Gerenciar inputs do usuário
  • Fazer chamadas HTTP

Explicação: O método ngOnInit() é um lifecycle hook chamado após a criação do componente, ideal para inicializar propriedades, fazer chamadas a serviços e outras inicializações necessárias. A destruição do componente é tratada por ngOnDestroy(), e embora seja comum fazer chamadas HTTP em ngOnInit(), essa não é sua única finalidade.

8. Qual comando instala o Angular CLI globalmente?

  • npm install -g @angular/cli
  • ng install angular
  • npm add angular-cli
  • yarn global add angular
  • npm i angular

Explicação: O comando npm install -g @angular/cli instala globalmente o pacote @angular/cli via npm, tornando o comando ng disponível em todo o sistema. O flag -g indica uma instalação global. As outras alternativas não são comandos válidos para instalar a CLI do Angular.

9. Qual é a função do async pipe?

  • Tratar Promises/Observables no template
  • Validação assíncrona
  • Chamadas HTTP
  • Gerenciamento de estado
  • Compartilhamento de dados

Explicação: O async pipe (| async) automaticamente inscreve/unsubscribe em Observables ou Promises no template, facilitando o trabalho com dados assíncronos. Ele lida com o gerenciamento do ciclo de vida da inscrição, evitando memory leaks e simplificando o código.

10. Como declarar um módulo raiz no Angular?

  • @NgModule({ declarations: [...] })
  • @Component({ providers: [...] })
  • @Directive({ selector: [...] })
  • @Injectable({ providedIn: [...] })
  • @Pipe({ name: [...] })

Explicação: O decorador @NgModule é usado para declarar um módulo Angular, organizando componentes, diretivas, pipes e serviços em blocos coesos de funcionalidade. A propriedade declarations é usada para listar os componentes, diretivas e pipes que pertencem ao módulo.
