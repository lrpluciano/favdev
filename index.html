Sistemas de Arquivos

O Sistema de Arquivos é o componente do Sistema Operacional responsável pelo gerenciamento, armazenamento e recuperação de dados a longo prazo.

O que isso significa na prática?

"Pensem no Sistema Operacional como um tradutor universal. Quando você clica em 'Salvar', o computador (hardware) só entende de magnetizar setores em um disco. O SO faz a ponte: ele pega seu documento e organiza isso no disco de uma forma que, amanhã, ele consiga remontar tudo."

Requisitos Essenciais

  • Grande Volume: Capacidade de armazenar enormes quantidades de dados.
  • Persistência: A informação sobrevive ao desligamento do computador.
  • Concorrência: Múltiplos usuários acessando o mesmo arquivo com segurança.

Entendendo a Persistência

"A RAM é a memória de curto prazo, como a nossa memória de trabalho. O Sistema de Arquivos é o 'papel e caneta'. Se a luz cair, o que estava na RAM some. O que foi salvo no Sistema de Arquivos fica lá para sempre (até você apagar)."

Estrutura e Tipos de Arquivos

Os arquivos podem ser vistos de diferentes formas pelo sistema operacional.

1. Estrutura Lógica

Sequência de Bytes

O SO não impõe estrutura. É um fluxo contínuo.

Registros Fixos

Arquivo dividido em blocos iguais. Comum em mainframes.

Árvore de Registros

Organização complexa por chaves. Ideal para bancos de dados.

Por que "Sequência de Bytes" venceu?

"Notem que quase tudo hoje é 'Byte Stream'. Por quê? Flexibilidade! Se o SO dita que o arquivo precisa ser dividido em blocos de 100 caracteres, eu não consigo salvar um nome de 101 letras. Com bytes, o SO diz: 'Aqui está o espaço, escreva o que quiser'. É por isso que seu Word, Excel e Fotos funcionam no mesmo disco."

2. Tipos de Arquivos Regulares

TipoCaracterísticasExemplos
ASCII / Texto Legível por humanos, portável. .txt, .html, .py
Binário Estrutura interna, requer software específico. .exe, .pdf, .jpg

O perigo de abrir Binários

"Se você tentar abrir um arquivo .exe ou .jpg no Bloco de Notas, vai ver um monte de caracteres estranhos. Isso acontece porque o Bloco de Notas espera ASCII (letras), mas o arquivo é binário (números para o processador). É como tentar ler partituras musicais como se fossem um romance."

3. Métodos de Acesso

  • Sequencial: Lê do início ao fim.
  • Direto: Pula para um registro específico.
  • Indexado (Chave): Usa um índice para busca rápida.

Analogia: O Índice do Livro

"Acesso Sequencial é ler o livro todo para achar uma frase. Acesso Indexado é abrir o livro na última página onde tem o índice, procurar o termo, ver a página e ir direto para lá. É o método mais rápido para bancos de dados gigantes."

Sistemas de Diretórios

Se arquivos são as informações, os diretórios são os organizadores.

Analogia da Biblioteca

"Imaginem uma biblioteca sem estantes. Todos os livros jogados no chão de um salão gigante. Para encontrar um livro, você teria que vasculhar um por um. Isso seria um sistema sem diretórios! Os diretórios são as estantes que organizam por assunto."

Como explicar no GPS?

"Caminho Absoluto é dar o endereço completo: 'Rua das Flores, 123, São Paulo'. Caminho Relativo é dar instruções de onde você está: 'Vire à direita na próxima esquina'. O computador usa o relativo para ser mais rápido, sem precisar escrever o endereço inteiro toda hora."

Implementação e Alocação

Como o SO decide onde gravar os dados no disco físico?

O desafio do Estacionamento

"O disco é um estacionamento com vagas numeradas (blocos). Quando você chega com um carro grande (arquivo), onde estaciona? Se as vagas 1, 2 e 3 estiverem ocupadas, você precisa ir para a 4, 5 e 6. O SO precisa anotar exatamente onde cada parte do carro estacionou."

Comparativo: Alocação no Disco
1. Alocação Contígua (Tudo junto):

O arquivo ocupa blocos seguidos. Rápido, mas difícil de encaixar.

A1
A2
A3
4
5
2. Alocação Encadeada (Espalhada):

Blocos espalhados, apontando uns para os outros.

A1
2
A2
A3

Métodos Principais

Contígua

Blocos sequenciais.
Problema: Fragmentação externa (buracos no disco).

Encadeada

Blocos ligados.
Problema: Leitura lenta se o arquivo estiver muito espalhado.

FAT / I-nodes

Usa tabelas de índice.
Solução moderna que balanceia velocidade e espaço.

O milagre dos I-nodes

"No Linux, cada arquivo tem um 'cartão de visita' chamado I-node. Esse cartão não guarda o conteúdo, mas sim uma lista de endereços: 'O conteúdo desse arquivo está nos blocos 10, 22 e 50'. Isso permite que o arquivo esteja espalhado pelo disco todo, mas para você parece que está todo junto na tela."

Segurança e Proteção

Proteger arquivos é garantir o acesso correto.

A regra de ouro: Menor Privilégio

"Segurança não é só colocar senha. É dar acesso apenas a quem precisa. Se todo mundo no escritório tiver a chave do cofre do diretor, é questão de tempo para dar problema. No SO, se todos podem 'escrever' no arquivo de sistema, alguém vai apagar sem querer."

Os 3 Pilares (CIA Triad)

Confidencialidade

Só autorizados veem. (Diário com cadeado)

Integridade

Ninguém alterou. (Carta lacrada)

Disponibilidade

Está lá quando precisar. (Hospital 24h)

Mecanismos de Proteção

MecanismoComo funciona
SenhaChave única para o arquivo.
GruposPermissão por departamentos (RH, TI).
ACLLista detalhada: "João lê, Maria escreve".

Ameaças Reais

"Não se esqueçam do fator humano. O firewall pode ser perfeito, mas se um funcionário clicar em um link de 'Prêmio Resgatado' (Phishing) e baixar um vírus, ele vai criptografar todos os arquivos. A segurança começa pela conscientização do usuário."
