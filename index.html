Bem-vindo à Oficina de Modelagem

Neste módulo, focaremos no desenvolvimento de competências práticas para a modelagem de sistemas utilizando a Linguagem de Modelagem Unificada (UML). Especificamente, trabalharemos com o Diagrama de Sequência.

Qual o objetivo?

O Diagrama de Sequência é fundamental para visualizar como os objetos de um sistema interagem entre si ao longo do tempo. Ele não mostra apenas quem faz o quê, mas quando e em qual ordem as mensagens são trocas.

Para realizar a atividade proposta neste encontro, você precisará transformar um cenário textual em uma representação visual lógica. Este site serve como seu guia de estudo para revisar os conceitos necessários antes de colocar a mão na massa.

Navegue pelas abas acima para revisar os elementos do diagrama, entender o método de extração de informações do texto e, finalmente, acessar o enunciado da sua atividade prática.

Elementos do Diagrama de Sequência

Para modelar corretamente, você deve dominar a sintaxe básica da UML para este tipo de diagrama. Revise os componentes abaixo:

1. Lifelines (Linhas de Vida)

Representam os participantes da interação. Podem ser atores (usuários) ou objetos do sistema (Interface, Controlador, Entidades). São desenhadas como um retângulo no topo com uma linha tracejada vertical descendente.

Dica: Procure pelos substantivos no cenário para identificar possíveis lifelines.

2. Messages (Mensagens)

Representam a comunicação entre as lifelines. Indicam que uma operação está sendo solicitada.

  • Síncrona (Seta cheia): O emissor espera uma resposta antes de continuar.
  • Assíncrona (Seta aberta): O emissor não espera resposta imediata.
  • Retorno (Linha tracejada): Indica o retorno de controle ou dados.

3. Activation Bars (Barras de Ativação)

Retângulos estreitos desenhados sobre a linha de vida. Indicam o período durante o qual o objeto está executando uma ação ou está ativo no processo.

Atenção: A ordem vertical das mensagens representa a ordem cronológica dos eventos. O tempo flui de cima para baixo.

Como transformar Texto em Diagrama

Modelar não é apenas desenhar, é interpretar. Siga este fluxo lógico ao ler o cenário da atividade:

  1. Identifique o Gatilho: O que inicia o processo? (Ex: Um clique, um evento de tempo, uma chegada de dados).
  2. Mapeie os Participantes: Quem está envolvido? Geralmente seguimos o padrão de camadas:
    • Boundary (Interface com o usuário)
    • Control (Lógica de negócio/Coordenador)
    • Entity (Dados/Informação)
  3. Extraia as Ações: Procure por verbos no texto. Cada ação significativa geralmente se traduz em uma mensagem entre objetos.
  4. Defina os Retornos: Para cada consulta ou solicitação, há uma resposta? O sistema precisa confirmar algo ao usuário?

Exemplo Genérico (Não é a atividade)

Imagine um cenário de Consulta de Saldo:

  • Texto: "O cliente solicita o saldo na tela."
  • Modelagem: Cliente envia mensagem solicitarSaldo() para Tela.
  • Texto: "O sistema busca os dados no banco."
  • Modelagem: Controlador envia mensagem buscarDados() para Banco.

Cenário: Sistema de Controle de Cinema

Considere um sistema de controle de cinema no qual um funcionário é responsável por realizar a venda de ingressos aos clientes.

  1. O processo inicia quando o funcionário seleciona a opção de venda de ingressos na interface do sistema.
  2. A interface solicita ao controlador do sistema que execute a operação de venda.
  3. O controlador deve consultar todas as sessões disponíveis para exibição, obtendo informações sobre a sala onde cada sessão ocorrerá e o filme que será apresentado.
  4. Após reunir as informações, o sistema retorna à interface uma lista contendo todas as sessões disponíveis para escolha.
  5. O funcionário seleciona, na interface, a sessão desejada e confirma a operação.
  6. Ao receber a confirmação, o controlador solicita que o sistema gere o ingresso correspondente, associando a escolha do funcionário à sessão selecionada.
  7. O sistema cria o ingresso com os dados necessários e retorna essa informação ao controlador.
  8. O controlador solicita à interface que exiba a confirmação da emissão do ingresso.

Entregáveis Esperados

  • Diagrama de Sequência completo.
  • Lifelines representando corretamente os objetos (Interface, Controlador, Entidades de Sessão/Ingresso, etc.).
  • Mensagens ordenadas cronologicamente conforme o fluxo descrito.
  • Coerência com o Diagrama de Classes desenvolvido no encontro anterior.
